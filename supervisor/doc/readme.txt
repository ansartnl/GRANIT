Supervisor

Supervisor состоит из 2-х модулей:
1. Сервер - используются для запуска описанных в конфигурации процессов/сервисов
2. Обозревателя - осуществляющего мониторинг запущенных процессов/сервисов, а также 
управления ими и группами запуска

Внимание!!! Настройка процессов требует указание непосредственного бинарного файла 
приложения в тэге <path> конфигурационного файла. Скрипт указывать нельзя, иначе 
мониторинг процесса не будет выполнен (не будет работать перезапуск).

Работа сервера:
Сервер ожидает соединение с базой данных и в момент наличия (если для группы настроены несколько 
соединений с разными БД - то будет ожидаться подключение ко всем БД в первую очередь) начинает 
запускать процессы/сервисы для текущей группы. Процесс/сервис имеет свойство перезапуска. Если 
перезапуск установлен и процесс/сервис 'крашится', то перезапуск выполняется опять же после 
проверки соединения с БД (если такая с процессом/сервисом связана). В том случае если соединения 
с БД отсутствует - процесс/сервис ожидает и не перезапускается.

В сервере предопределены следующие специальные обработчики:
- CWP процесс - тэг <cwp>true<cwp> в настройках процессов
- FDP процесс/сервис - тэг <fdp>true<fdp> в настройках процессов/сервисов
- FMTP процесс/сервис - тэг <fmtp>true<fmtp> в настройках процессов/сервисов

Windows:
Для работы сервера под Windows OC никаких дополнительных, кроме настройки конфигурационного 
файла, настроек не требуется. При необходимости и в случае использовании сервисов - требуется 
их регистрация.

Linux:
Для запуска сервера требуется его доступ к переменным окружения всех библиотек, которые будут 
использоваться процессами/сервисами. Этого можно достичь либо с помощью регистрации переменной 
PATH и  LD_LIBRARY_PATH в файле .bashrc для текущего пользователя. Либо стартовать сервер 
с помощью скрипта, в котором как минимум переменная LD_LIBRARY_PATH должна быть определена и 
экспортирована для указания пути к библиотекам (SupervisorServer.sh).

Регистрация сервиса выполняется командой:
<service-name> -i

Отмена регистрации сервиса выполняется командой:
<service-name> -u

Файл конфигурации клиента:
<simplified_view>false</simplified_view> - режим запуска 'Обозревателя'. Если true - то запуск 
выполняется с интерфейсом в урезанном виде

Файл конфигурации сервера:
<groups> - группы запуска. Имена тэгов children-ов (конкретных групп) не имеет значения и может 
быть любым.
<anyTagNameGroup visualName="CWP Group" id="CWP"> - обязательным является атрибут 'id' - определяет 
уникальное имя группы. 'visualName' - нужен для визуального представления группы в окне 'Обозревателя'.

Каждая группа содержит произвольное количество процессов с произвольным именем тэга, но обязательным 
текстом, который является атрибутом 'id' из настроек процесса/сервиса:
<process1 order="1">STCA-Gr1-1</process1>
Атрибут 'order' у процесса означает порядок его пуска, но не гарантирует порядок его запуска. Этот атрибут 
может быть проигнорирован.

Настройка процессов/сервисов выделена в отдельные группы тэгов:
- процессы: <processes>
- сервисы: <services>

Различие только в 2-х тэгах:
- <name> - для сервисов и означает имя сервиса, который будет запускаться
- <path> - для процесса - путь к бинарному файлу приложения.
Остальные параметры одинаковые и могут отсутствовать, если в них нет необходимости (либо быть пустыми).
Общие параметры:
<arguments> - параметры командной строки
<database> - 'id' атрибут БД, от которой будет зависеть процесс/сервис
<description> - необязательное описание процесса/сервиса
<restart> - параметр, указывающий необходимость перезапуска в случае падения: true
<visualName> - имя процесса/сервиса, отображаемое в окне 'Обозреватель'
<log_path> - путь к файлу лога, для возможности удаленного просмотра лога 'Обозревателем'
<config_path> - путь к файлу конфигурации процесса/сервиса - для внутренней обработки конкретными 
обработчиками сервера, например, FMTP сервис
Смотри описание специальных тэгов для процессов/сервисов выше: <cwp> <fdp> <fmtp>. Для этих 
процессов/сервисов предусмотрены специализированные (захардкоженые) обработчики (для FDP например - это 
получение спец статусов).

Для того, чтобы процесс/сервис мог участвовать в группах - для него должен быть задан атрибут 'id':
    <processes>
        <assistant id="Assistant">

Настройка баз данных выполенена через группу БД - <databases>:
<database1 id="Coala">
	<driver>QOCI</driver>
	<host>192.20.1.62</host>
	<name>XE</name>
	<password>aero</password>
	<port>1521</port>
	<user>aero</user>
Обязательным атрибутом для БД является 'id'. Этот уникальный атрибут используется в параметре 
процесса/сервиса для связи с данной БД (<database>).

Запуск клиента возможен с использованием стилей и палитры пользователя, которые настраиваются 
в приложении styleEditor. Стили пользователя хранятся в файле с расширением 'es'. Файл может быть 
подключен следующим образом:
- находится в папке ../etc/%имя-приложения%.es
- либо абсолютным путем в командной строке с опцией --extendedStyle (UNIX style option)
